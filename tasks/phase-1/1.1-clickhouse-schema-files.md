## Goal

Land all ClickHouse DDL the collector needs as numbered SQL files in `apps/collector/db/migrations/`. The migration runner (task 1.2) reads them in filename order. After this task, a fresh ClickHouse instance can be brought up to schema with one command.

## Context

- ClickHouse is sealed inside the collector — only the collector talks to it.
- Schema reference is in `docs/reference/clickhouse-schema.md`. **Copy it byte-for-byte** unless you have a justifying reason; deviations require a docs update in the same PR.
- Tables: `events` (raw), `events_buffer` (Buffer engine in front), 5 materialized views, `fx_rates` dictionary, `_migrations` tracker.

## Acceptance criteria

- All 9 SQL files exist under `apps/collector/db/migrations/`.
- Each file is idempotent (`CREATE TABLE IF NOT EXISTS`, `CREATE MATERIALIZED VIEW IF NOT EXISTS`, etc.).
- The DDL matches `docs/reference/clickhouse-schema.md` exactly.
- Running them in order against a fresh CH 24.x instance succeeds without errors.
- A trivial smoke test (`docker compose up -d clickhouse && for f in apps/collector/db/migrations/*.sql; do clickhouse-client < $f; done`) reaches a clean state.

## Implementation notes

- ORDER BY for `events`: `(project_id, event_name, ts)`. Every dashboard query starts with these three.
- Partitioning: `toYYYYMM(ts)`. Lets TTL drop a partition atomically.
- TTL: `toDateTime(ts) + INTERVAL 13 MONTH` on the raw `events` table only. MVs are tiny and kept indefinitely.
- Buffer engine parameters: `Buffer(currentDatabase(), events, 16, 5, 30, 10000, 100000, 10000000, 100000000)`. Don't tune yet; defaults are fine for v1.
- For the FX dictionary, the `SOURCE(HTTP(...))` URL is `http://collector:8000/_internal/fx-rates` — that's the docker-compose service name. Production deploy will need a value override; not our concern in this task.
- `_migrations` tracker is a plain `MergeTree` keyed by filename. Migration runner inserts a row on success.

## Tests

This task has no runtime code, so no Vitest. Test plan (manual):

1. `docker compose up -d clickhouse`
2. Wait for healthy.
3. `for f in apps/collector/db/migrations/*.sql; do docker compose exec -T clickhouse clickhouse-client --multiquery < "$f"; done`
4. `docker compose exec clickhouse clickhouse-client -q "SHOW TABLES"` should list events, events_buffer, mv_*.
5. `docker compose exec clickhouse clickhouse-client -q "DESC events"` should match the column list in the reference doc.

Document the manual smoke test in the PR description.
