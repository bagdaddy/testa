## Goal

Bring Phase 1 to comfortable test coverage. Each unit's spec was sketched in its own task; this task is the **integration** test that proves the chain works end-to-end (edge worker NOT involved — we simulate it by calling `/_ingest` ourselves with a real HMAC).

## Acceptance criteria

- `bun test` passes locally with redis + clickhouse from `docker-compose`.
- CI's `test-collector` job is green.
- An integration test exists that:
  1. Starts the collector HTTP server in-process.
  2. Starts the consumer loop in a goroutine-equivalent (Bun worker thread or just `Promise.race` with shutdown signal).
  3. Posts a signed batch of 50 events to `/_ingest`.
  4. Polls CH for `count() = 50` (with timeout).
  5. Asserts the rows match the input shape, modulo defaults.
- Test file under 200 LOC; covers happy-path AND the three primary failure modes (bad sig, replay window, schema fail).
- Coverage on `apps/collector/src/auth/`, `apps/collector/src/ingest/`, `apps/collector/src/consumer/` ≥ 80% lines.

## Implementation notes

### HMAC test vectors

Pin one or two known HMAC vectors in `auth/__tests__/hmac.test.ts`:

```ts
import { describe, expect, it } from 'bun:test';
import { verifyHmac } from '../hmac.ts';

const SECRET = 'test-secret-for-vectors-only-do-not-use';

describe('verifyHmac', () => {
  it('accepts a known-good vector', () => {
    const body = '{"signed_at":1730902400123,"events":[]}';
    // generated via openssl dgst -sha256 -hmac (pin the actual hex here)
    const sig = '<expected-hex>';
    expect(verifyHmac(body, sig, SECRET)).toBe(true);
  });

  it('rejects altered body', () => {
    expect(verifyHmac('{"events":[]}', '<expected-hex>', SECRET)).toBe(false);
  });

  it('rejects wrong-length signature', () => {
    expect(verifyHmac('body', 'aabb', SECRET)).toBe(false);
  });

  it('rejects non-hex signature', () => {
    expect(verifyHmac('body', 'not-hex', SECRET)).toBe(false);
  });
});
```

### Integration test sketch

```ts
// apps/collector/src/__tests__/integration.test.ts
import { describe, expect, it, beforeAll, afterAll } from 'bun:test';
import { createHmac } from 'node:crypto';
import { runLoop } from '../consumer/loop.ts';
import { state } from '../consumer/state.ts';
import server from '../index.ts';
import { query } from '../db/clickhouse.ts';
import { config } from '../config.ts';

describe('phase 1 integration', () => {
  let consumerPromise: Promise<void>;

  beforeAll(async () => {
    consumerPromise = runLoop();
  });

  afterAll(async () => {
    state.shutdown = true;
    await consumerPromise;
  });

  it('signed batch lands in CH within 10s', async () => {
    const events = makeEvents(50);
    const body = JSON.stringify({ signed_at: Date.now(), events });
    const sig = createHmac('sha256', config.ingest.sharedSecret).update(body).digest('hex');

    const res = await server.fetch(
      new Request('http://test.local/_ingest', {
        method: 'POST',
        headers: { 'content-type': 'application/json', 'x-edge-signature': sig },
        body,
      }),
    );
    expect(res.status).toBe(204);

    // Poll CH
    const expected = events.length;
    const deadline = Date.now() + 10_000;
    let got = 0;
    while (Date.now() < deadline) {
      const rows = await query<{ c: number }>(
        "SELECT count() AS c FROM events WHERE event_id IN ({ids:Array(UUID)})",
        { ids: events.map((e) => e.event_id) },
      );
      got = rows[0]?.c ?? 0;
      if (got === expected) break;
      await new Promise((r) => setTimeout(r, 250));
    }
    expect(got).toBe(expected);
  }, 15_000);
});
```

### Test data hygiene

- Use unique `project_id` per test run (e.g. `project_id = 1_000_000 + Date.now() % 1000`) so tests can't pollute each other.
- After each test, optionally delete via `ALTER TABLE events DELETE WHERE project_id = ...` — but with the test pid prefix, leakage is harmless.
