## Goal

Implement `POST /_ingest`. Verifies HMAC + replay window, Zod-validates the batch, XADDs each event onto Redis Stream `events`. Returns 204.

## Context

This is the **only** trust boundary between the public-facing edge worker and our private infra. Get HMAC verification right.

The HMAC contract is fully specified in `docs/reference/hmac-protocol.md`. **Read it before coding.**

The event shape is fully specified in `docs/reference/event-shape.md`. **Read it before coding.**

## Acceptance criteria

- Valid signed batch → 204 No Content. Each event lands on Redis Stream `events` (verifiable with `XLEN events`).
- Missing or malformed `X-Edge-Signature` → 401.
- Wrong signature → 401.
- `signed_at` outside ±5 min window → 400 with reason "replay window exceeded".
- Body fails Zod validation → 400 with details.
- Empty batch (`events: []`) → 204 (no-op).
- HMAC verification uses constant-time comparison (`timingSafeEqual` or equivalent).
- Body is read once as raw text and reused; no double-parse.

## Implementation notes

### `auth/hmac.ts`

Pure helper:

```ts
import { createHmac, timingSafeEqual } from 'node:crypto';

export function verifyHmac(rawBody: string, signatureHex: string, secret: string): boolean {
  if (!signatureHex || !/^[0-9a-f]+$/i.test(signatureHex)) return false;
  const expected = createHmac('sha256', secret).update(rawBody).digest();
  let provided: Buffer;
  try {
    provided = Buffer.from(signatureHex, 'hex');
  } catch {
    return false;
  }
  if (provided.length !== expected.length) return false;
  return timingSafeEqual(expected, provided);
}
```

### `auth/middleware.ts`

Hono middleware that:

1. Reads `c.req.header('x-edge-signature')`.
2. Reads raw body via `await c.req.text()` (Hono caches; avoid re-reading).
3. Verifies HMAC. On fail: `return c.text('invalid signature', 401)`.
4. Stashes raw body on context: `c.set('rawBody', raw)`.
5. Calls `next()`.

```ts
import { createMiddleware } from 'hono/factory';
import { config } from '../config.ts';
import { verifyHmac } from './hmac.ts';

declare module 'hono' {
  interface ContextVariableMap {
    rawBody: string;
  }
}

export const requireEdgeSignature = createMiddleware(async (c, next) => {
  const sig = c.req.header('x-edge-signature') ?? '';
  const raw = await c.req.text();
  if (!verifyHmac(raw, sig, config.ingest.sharedSecret)) {
    return c.text('invalid signature', 401);
  }
  c.set('rawBody', raw);
  await next();
});
```

### `ingest/route.ts`

```ts
import { Hono } from 'hono';
import Redis from 'ioredis';
import { config } from '../config.ts';
import { requireEdgeSignature } from '../auth/middleware.ts';
import { ingestBatchSchema } from './schema.ts';

const redis = new Redis(config.redis.url);

export const ingestRoutes = new Hono();

ingestRoutes.post('/_ingest', requireEdgeSignature, async (c) => {
  const raw = c.get('rawBody');
  let body: unknown;
  try {
    body = JSON.parse(raw);
  } catch {
    return c.text('invalid JSON', 400);
  }

  const parsed = ingestBatchSchema.safeParse(body);
  if (!parsed.success) {
    return c.text(`schema error: ${parsed.error.message}`, 400);
  }
  const { signed_at, events } = parsed.data;

  const skew = Math.abs(Date.now() - signed_at);
  if (skew > config.ingest.replayWindowSec * 1000) {
    return c.text('replay window exceeded', 400);
  }

  if (events.length === 0) {
    return c.body(null, 204);
  }

  // Pipeline SETNX (for dedup-allowed events) + XADD for throughput.
  // Dedup runs only for event_names in INGEST_DEDUP_EVENT_NAMES (default `['purchase']`)
  // — see docs/architecture/02-collector.md § Idempotency for the rationale.
  const dedupNames = config.ingest.dedupEventNames;
  const pipe = redis.pipeline();
  const dedupKeys: { idx: number; key: string }[] = [];
  for (let i = 0; i < events.length; i++) {
    const ev = events[i]!;
    if (dedupNames.has(ev.event_name)) {
      const key = `event:seen:${ev.event_id}`;
      pipe.set(key, '1', 'EX', 600, 'NX');
      dedupKeys.push({ idx: i, key });
    }
  }
  const setReplies = await pipe.exec();
  // For each dedup-checked event, if SET returned null (already existed), skip its XADD.
  const skipIdx = new Set<number>();
  if (setReplies) {
    for (let j = 0; j < dedupKeys.length; j++) {
      const reply = setReplies[j]?.[1];
      if (reply === null) skipIdx.add(dedupKeys[j]!.idx);
    }
  }

  const pipe2 = redis.pipeline();
  for (let i = 0; i < events.length; i++) {
    if (skipIdx.has(i)) continue;
    pipe2.xadd(config.redis.streamKey, '*', 'event', JSON.stringify(events[i]));
  }
  await pipe2.exec();

  return c.body(null, 204);
});
```

Note: we serialize each event to a single field `event` rather than spreading every column. Simplifies the consumer and keeps stream entries compact. Consumer does `JSON.parse(fields.event)`.

### Idempotency / dedup

Same `event_id` may arrive multiple times (pixel IDB outbox retries, edge retries on 5xx). Dedup mechanism: `SET event:seen:<event_id> 1 EX 600 NX` BEFORE `XADD`. If the SET returns `nil`, this event is a duplicate — skip the XADD entirely. Applied **only** to event names in `INGEST_DEDUP_EVENT_NAMES` (default `['purchase']`). Other event types are mostly immune to duplication via `uniqExact` aggregations in CH; defer their dedup to a future env-var change.

`config.ts` needs a new field:

```ts
INGEST_DEDUP_EVENT_NAMES: z.string().default('purchase')
  .transform((s) => new Set(s.split(',').map((x) => x.trim()).filter(Boolean))),
```

Memory is bounded: at 250 ev/s × ~10% purchase mix × 600 s = ~15,000 keys ≈ ~1 MB Redis. Negligible.

> **Earlier doc drafts** described a "deterministic stream entry IDs" approach (`XADD events <event_id>-<ts_ms>`). That doesn't actually work because Redis Streams require strictly monotonic IDs across the whole stream. `SETNX`-before-`XADD` is the correct pattern.

## Tests

`apps/collector/src/ingest/__tests__/route.test.ts`:

- Build a valid signed body; assert 204 + `XLEN events` increases by N.
- Tamper signature → 401.
- Tamper body → 401.
- Bump `signed_at` 10 min in the past → 400 "replay window exceeded".
- Empty events → 204, no XADDs.
- Schema fail (missing `event_id`) → 400.
- POST a `purchase` twice with the same `event_id` → only one row in stream (dedup works).
- POST a `page_view` twice with the same `event_id` → both rows in stream (dedup not applied to non-allow-listed names).

Use real Redis (CI already has it as a service container). Don't mock — too easy to get wrong.

## Out of scope

- Per-project rate limiting — explicitly NOT applied. Per `docs/architecture/05-rollout.md` § Edge worker deployment model, customer isolation comes from per-customer worker deployments + crobot's lead quotas, not technical caps.
- Compression (request/response). Keep it simple for v1.
- Multi-secret rotation. Single `INGEST_SHARED_SECRET` for now.
- Metrics / observability. Add separately when collector is otherwise stable.
- Extending the dedup allow-list beyond `purchase`. Operators can add `email_submit` (or others) by changing `INGEST_DEDUP_EVENT_NAMES` env without code changes.
