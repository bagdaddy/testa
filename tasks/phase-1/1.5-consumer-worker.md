## Goal

A long-running Bun process that drains Redis Stream `events` and INSERTs batches into ClickHouse `events_buffer`. Single instance for v1; consumer-group-aware so we can scale later without changing the wire protocol.

## Acceptance criteria

- `bun run dev:consumer` starts the loop and connects to Redis + CH.
- On startup, ensure consumer group `collector-writers` exists (`XGROUP CREATE events collector-writers $ MKSTREAM` — ignore "BUSYGROUP" error).
- Reads up to `CONSUMER_BATCH_SIZE` (default 1000) messages OR blocks `CONSUMER_FLUSH_INTERVAL_MS` (default 5000), whichever first.
- Parses each message's `event` field, builds rows for `events_buffer`, INSERTs in one batch.
- On success: XACK all messages.
- On CH 5xx: don't XACK. Sleep with exp backoff (1s, 2s, 4s, 8s, max 30s) and retry the same batch with `XREADGROUP ... 0` (re-read pending entries).
- On CH 4xx (schema mismatch): log loud, XACK to drop, report. **Don't loop forever on poison pill.**
- On SIGTERM / SIGINT: complete the in-flight INSERT, XACK, close clients, exit 0.
- Pending recovery: every 60s, XCLAIM messages in pending list older than 30s (handles consumers that crashed mid-batch).

### Main loop

```ts
// apps/collector/src/consumer/loop.ts (sketch)
import Redis from 'ioredis';
import { config } from '../config.ts';
import { insertEvents } from '../db/clickhouse.ts';
import { state } from './state.ts';

const STREAM = config.redis.streamKey;
const GROUP = config.redis.consumerGroup;
const CONSUMER = config.redis.consumerName;

export async function runLoop(): Promise<void> {
  const redis = new Redis(config.redis.url);
  await ensureGroup(redis);

  let backoff = 0;

  while (!state.shutdown) {
    if (backoff > 0) {
      await sleep(backoff);
      backoff = Math.min(backoff * 2, 30_000);
    }

    // First drain any pending messages from a previous crashed consumer
    const pending = await redis.xreadgroup(
      'GROUP', GROUP, CONSUMER,
      'COUNT', config.consumer.batchSize,
      'STREAMS', STREAM, '0',
    );
    if (pending && (pending as any).length) {
      const ok = await processBatch(pending);
      if (!ok) { backoff = backoff || 1000; continue; }
    }

    // Then read new messages, blocking up to flushInterval
    const fresh = await redis.xreadgroup(
      'GROUP', GROUP, CONSUMER,
      'COUNT', config.consumer.batchSize,
      'BLOCK', config.consumer.flushIntervalMs,
      'STREAMS', STREAM, '>',
    );
    if (fresh && (fresh as any).length) {
      const ok = await processBatch(fresh);
      if (!ok) { backoff = backoff || 1000; continue; }
    }
    backoff = 0;
  }

  await redis.quit();
}

async function processBatch(streamResult: any): Promise<boolean> {
  // streamResult = [[STREAM, [[id, [field, value, ...]], ...]]]
  const entries = streamResult[0][1] as Array<[string, string[]]>;
  const ids: string[] = [];
  const rows: object[] = [];

  for (const [id, fields] of entries) {
    ids.push(id);
    const eventIdx = fields.indexOf('event');
    if (eventIdx === -1) continue;
    try {
      rows.push(JSON.parse(fields[eventIdx + 1] as string));
    } catch (e) {
      console.error('[consumer] bad event payload', id, e);
      // still XACK to drop poison
    }
  }
  if (rows.length === 0) {
    await xackBatch(ids);
    return true;
  }

  state.inflight = { ids, events: rows };
  try {
    await insertEvents(rows);
    await xackBatch(ids);
    state.inflight = null;
    return true;
  } catch (err: any) {
    state.inflight = null;
    const status = err?.code ?? err?.status;
    if (status >= 400 && status < 500) {
      console.error('[consumer] CH 4xx — dropping poison batch', err);
      await xackBatch(ids);   // drop
      return true;
    }
    console.error('[consumer] CH 5xx — will retry batch', err);
    return false;
  }
}
```

(`ensureGroup`, `xackBatch`, `sleep` left as small helpers.)

### Main entry

```ts
// apps/collector/src/consumer/main.ts
import { state } from './state.ts';
import { runLoop } from './loop.ts';
import { close as closeCh } from '../db/clickhouse.ts';

async function main(): Promise<void> {
  console.log('[consumer] starting');

  process.on('SIGTERM', () => { state.shutdown = true; });
  process.on('SIGINT', () => { state.shutdown = true; });

  try {
    await runLoop();
  } finally {
    await closeCh();
    console.log('[consumer] exited cleanly');
  }
}

void main();
```

### Pending recovery (XCLAIM)

Add a separate timer (every 60s):

```ts
const idle = await redis.xpending(STREAM, GROUP, '-', '+', 100);
const stuck = idle.filter((e) => e.idle > 30_000);
if (stuck.length) {
  await redis.xclaim(STREAM, GROUP, CONSUMER, 30_000, ...stuck.map((e) => e.id));
}
```

Belts-and-braces; the main loop's `0` read normally handles this when the same consumer reconnects, but a different consumer name could leave pending forever otherwise.

### CH row shape

`@clickhouse/client` `insert({ format: 'JSONEachRow' })` accepts an array of plain objects matching column names. The Zod-validated event already matches the `events` schema field-by-field, except:

- Default-fill missing optional fields (`referrer = ''`, `value_native = 0`, `currency = ''`, `order_id = ''`, `items_count = 0`, `props = {}`).
- Convert `ts` and `ingested_at` from Unix-ms numbers to CH `DateTime64(3)` strings (e.g. `'2026-05-05 17:30:00.123'`) — `@clickhouse/client` handles this if you pass the millisecond Date object, but verify against the docs.

Add a tiny `eventToRow(ev)` helper.

## Out of scope

- Multi-consumer scaling (consumer group is set up but only one process for v1).
- Cross-region replication.
- Per-event deduplication via Redis SET (add later if dupes show up; UUIDs make it unlikely).
- Backpressure to the producer. Stream length is bounded by ingest rate; CH should keep up.
