## Goal

Mirror the entire `window.Analytica.*` surface so 3.x customer integrations continue to work. Wire the legacy `eventEmitter` to the new internal event bus so customers' `Analytica.eventEmitter.on(...)` calls fire at the right lifecycle points.

## Acceptance criteria

- Every field listed in `docs/reference/legacy-globals-inventory.md` is present on `window.Analytica` after runtime loads, with the right initial value where applicable.
- `Analytica.eventEmitter` exposes `on`, `emit`, `_processHistoryForHandler`, plus history replay for late subscribers — same semantics as 3.6.
- `Analytica.cookies`, `.ses`, `.usr`, `.excl`, `.sent`, `.uuid` are *live mirrors* of the canonical state — when the new code mutates internal state, the legacy globals reflect it.
- `Analytica.isLoaded` flips `true` after first init cycle. Deprecated but kept.
- `Analytica.isRedirecting` is `true` for the duration of any redirect.
- `Analytica.pushEvent(name, data)` triggers the new track pipeline AND the legacy `goals[]` `createConversion` path.
- `Analytica.isNextApp` is detected via the same heuristics as 3.6.
- Every assertion in `legacy-globals.test.ts` passes.

## Implementation notes

The trick is keeping the legacy globals in sync without spreading mutation across many modules. Use a single `legacy.ts` that exports a "publish" function the new modules call after they update internal state:

```ts
// apps/pixel/src/runtime/legacy.ts
export function publishCookieAssignment(experimentId: number, variationId: number): void {
  window.Analytica.cookies[experimentId] = variationId;
}
export function publishSession(experimentId: number, expiry: number): void {
  window.Analytica.ses[experimentId] = expiry;
}
// etc.
```

Internal modules (cookies, traffic, apply, redirect) import these and call after their canonical update. The legacy module becomes the single thread of mutation onto `window.Analytica`, easy to audit.

For `eventEmitter`: implement same shape as 3.6 (history replay, WeakMap-based handler dedup). Internal `fireEvent('variation_applied', data)` from the apply module routes through this emitter.

    expect(window.Analytica.COOKIE_NAME).toBe('_testa_exp');
    // ... full enumeration
  });

  it('eventEmitter.on receives variation_applied', async () => {
    const handler = vi.fn();

  });

  it('cookies / ses / excl / etc. mutations are reflected', () => {
