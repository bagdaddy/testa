## Goal

Read/write the 5 cookies the pixel cares about, with consent-aware policies. Provides a single typed module other runtime code imports.

| Cookie | Set by | Max-Age | First-party? | Owner |
|---|---|---|---|---|
| `_testa_uuid` | Worker via `Set-Cookie` | 2y | depends on CNAME mode | edge worker writes; pixel reads |
| `_testa_ses` | Pixel `document.cookie` | 1h sliding | always | pixel |
| `_testa_exp_<id>` | Pixel `document.cookie` | 30d | always | pixel |
| `_testa_excl_<id>` | Pixel `document.cookie` | 30d | always | pixel |
| `_testa_user_<id>` | Pixel `document.cookie` | 30d | always | pixel |

## Acceptance criteria

- `getUuid()`, `getSession(experimentId)`, `getAssignment(experimentId)`, `getExclusion(experimentId)`, `getFirstSeen(experimentId)` return typed reads.
- `setSession(experimentId)`, `setAssignment(experimentId, variationId)`, `setExclusion(experimentId, value)`, `setFirstSeen(experimentId, ts)` write with the right TTLs.
- `clearAll()` resets every `_testa_*` cookie.
- Cookie names use the constants from `legacy-globals-inventory.md` (`COOKIE_NAME = '_testa_exp'`, `SESSION_COOKIE = '_testa_ses'`, etc.).
- Consent-denied: `setUuid` is no-op (cookie set with `Max-Age=0` to clear an existing one); the worker is the source of truth for `_testa_uuid` writes.
- Domain inference: shared mode → no `Domain=`; CNAME mode is set by the worker, pixel doesn't write `_testa_uuid`.
- `SameSite=Lax`, `Secure` everywhere we write.
- Tests cover round-trip read/write, TTL refresh on session activity, consent-denied no-op, name-prefix consistency.

## Implementation notes

```ts
// apps/pixel/src/runtime/cookies.ts (sketch)
const NAMES = {
  uuid: '_testa_uuid',
  ses:  '_testa_ses',
  exp:  '_testa_exp',
  excl: '_testa_excl',
  user: '_testa_user',
} as const;

const SESSION_LENGTH_MS = 60 * 60 * 1000;        // 1h sliding
const ASSIGNMENT_TTL_DAYS = 30;

export function getUuid(): string | null {
  return readCookie(NAMES.uuid);
}

export function setSession(experimentId: string): void {
  writeCookie(`${NAMES.ses}_${experimentId}`, String(Date.now()), {
    maxAgeSec: SESSION_LENGTH_MS / 1000,
    sameSite: 'Lax',
    secure: true,
  });
}
// ... similar for others
```

Don't reach into `Analytica.cookies` from here — that's the legacy globals module's job (3.12). This module is purely cookie I/O.

## Tests

- `setAssignment(17, 100)` then `getAssignment(17)` returns `'100'` (or `100`, depending on serialization choice — pick one and document).
- `setSession(17)` writes a cookie that's readable back; calling again refreshes the value.
- After `clearAll()`, every getter returns `null`.
- Consent-denied state set via the consent module: `setSession` becomes a no-op (assert via spy).
- Cookie names match the inventory's constants exactly.

## Out of scope

- Visitor ID generation / rotation — handled by the edge worker via `Set-Cookie`.
- `Analytica.cookies` legacy global mirroring — task 3.12.
- Consent state machine — task 3.4 owns it; this module reads its current state.
