## Goal

Entry point for the runtime bundle. Drains the loader's `_testa.q` queue, wires up real method implementations, fires `_testa.load()` once the first experiment-resolution cycle completes.

## Context

The runtime bundle is loaded via `<script defer>` after the loader. By the time it executes, the loader has populated `window._testa` with stubs and `window._testa.q` with any pre-runtime calls.

## Acceptance criteria

- `apps/pixel/src/runtime/index.ts` is the entry that the bundle's main module exports.
- On execution: imports + initializes cookies, consent, transport, audience, experiments, legacy, SPA, and lifecycle modules. (Each lives in its own subdir; this task wires them together.)
- After init: drains `window._testa.q` in arrival order, replacing each stub method with its live impl.
- After first experiment-resolution cycle finishes: calls `_testa._loaded()` to flip the load Promise.
- Re-entry on SPA route change: re-runs experiment evaluation (NOT the full init); `_testa.load()` is fire-once (does not re-resolve on every route change).
- Errors in module init are caught and logged as a `_pixel_health` event with a stack trace, so a single broken integration doesn't blank the page.

## Implementation notes

```ts
// apps/pixel/src/runtime/index.ts (sketch)
import { hydrate } from './lifecycle.ts';

try {
  hydrate();
} catch (err) {
  // emit _pixel_health with err details
}
```

`lifecycle.ts`:

```ts
export async function hydrate(): Promise<void> {
  initCookies();
  initConsent();
  initTransport();
  initLegacyGlobals();
  installSpaListeners();
  await initRuleEngine();          // loads project config from cfPrefill
  await runExperimentCycle();      // assigns + applies variations, fires experiment_view
  drainQueue();                     // replays _testa.q
  fireReady();                      // resolves _testa.load()
}
```

Each `init*` is from its own task (3.3 cookies, 3.4 consent, 3.5 SPA, etc.). This task ships the *wiring*, not the contents.

## Tests

happy-dom with a mocked `window._testa.q` populated before runtime runs:
- Stubs are replaced with live impls (`typeof _testa.track === 'function'` and not the queue stub).
- Queue is drained in order (`vi.spyOn(track)` → first arg is the first queued call).
- `_testa.load()` resolves after `fireReady()` is called.
- An error in `initCookies` does not throw out of `hydrate`; instead a `_pixel_health` event is emitted with the error message.

## Out of scope

- The actual contents of cookies / consent / audience / transport / experiments — each is its own task.
- SmartCode side (separate snippet shipped to customers).
