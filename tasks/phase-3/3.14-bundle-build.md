## Goal

Two bundles: a thin sync `loader.min.js` (~5 KB) and a deferred `runtime.min.js` (~30–40 KB). Runtime URL contains a content hash so the worker can cache it indefinitely; loader is small and re-served from KV per request.

## Acceptance criteria

- `pnpm --filter @testa-platform/pixel build` produces `dist/loader.min.js` and `dist/runtime-<hash>.min.js`.
- Loader is ESM, IIFE-wrapped (since it's served inline in the worker response). Bundle ≤6 KB minified+gzipped.
- Runtime is ESM, served via `<script defer>`. Bundle ≤45 KB minified+gzipped (target ≤35 KB; flag in PR if higher).
- Source maps are emitted and uploaded to Sentry (separate step, deferred).
- `<hash>` is the first 12 hex chars of `sha256(runtime contents)`.
- A small `upload-bundle.ts` script publishes both bundles to the CF KV `KV_INTEGRATION_BUNDLES` namespace (per `docs/architecture/05-rollout.md`).
- Build fails CI if the loader bundle exceeds 7 KB (hard cap) or runtime exceeds 60 KB (soft warning).

## Implementation notes

```js
// apps/pixel/esbuild.config.mjs (sketch)
import { build } from 'esbuild';
import { readFileSync } from 'node:fs';
import { createHash } from 'node:crypto';

await build({
  entryPoints: ['src/loader.ts'],
  bundle: true,
  minify: true,
  format: 'iife',
  outfile: 'dist/loader.min.js',
  target: ['es2020'],
});

const runtimeOutput = await build({
  entryPoints: ['src/runtime/index.ts'],
  bundle: true,
  minify: true,
  format: 'esm',
  outfile: 'dist/runtime.min.js',
  metafile: true,
  sourcemap: 'external',
  target: ['es2020'],
});

const hash = createHash('sha256')
  .update(readFileSync('dist/runtime.min.js'))
  .digest('hex')
  .slice(0, 12);
// move dist/runtime.min.js → dist/runtime-${hash}.min.js
```

The worker reads `KV.get('integration_bundle:4.0:loader')` + `KV.get('integration_bundle:4.0:runtime')` and constructs the response. Hash is in the runtime KV key so the worker can cache aggressively.
