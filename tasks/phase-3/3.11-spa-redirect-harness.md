## Goal

Tests that prove the new redirect engine works correctly under each SPA framework + version combo we want to support. Each fixture loads the pixel + runs a redirect experiment, then asserts the destination URL preserves query params and only fires once.

## Acceptance criteria

- Five fixture apps exist under `apps/pixel/test/spa-fixtures/`. Each is the smallest possible Next.js / RR / plain JS app that triggers an SPA navigation.
- Playwright spec at `apps/pixel/test/spa.spec.ts` runs each fixture through:
  1. Load fixture's `/page-A?utm_source=test&utm_campaign=q2`.
  2. Pixel detects experiment match, redirects to `/page-B`.
  3. Assert destination URL is `/page-B?utm_source=test&utm_campaign=q2`.
  4. Assert pixel's `_pixel_debug.redirects` log has exactly one entry.
  5. Click back; assert no re-redirect (location.replace was used, not assign).
- For Next.js fixtures specifically, also test the navigation-during-hydration case: pixel decides redirect during Next's hydration; assert no hydration mismatch warnings, no double-fire, query params intact.
- Run all fixtures in CI against `chromium` (Playwright default).

## Implementation notes

Each fixture is purely static once built — Playwright's `webServer` block in `playwright.config.ts` serves them. Fixtures don't need a real Node runtime.

For Next.js fixtures: build with `next export` (Next 13/14: `output: 'export'`). For Next 12 pages-router: `next build && next export`. The exported static HTML embeds the same JS that triggers the SPA bug in production.

The fixture HTML has `<script src="/loader.js"></script>` referencing the locally-built pixel from `apps/pixel/dist/`.

If any fixture catches a real bug in 3.10's redirect engine, that's the *whole point* — file it in `docs/KNOWN_ISSUES.md` and either fix in 3.10 or document the limitation.

## Tests

The spec itself IS the test. CI artifact uploads include screenshots of each redirect destination + the `_pixel_debug` log dump on failure.

## Out of scope

- Browser combinations beyond chromium (CI cost).
- Mobile viewports (separate spec later).
