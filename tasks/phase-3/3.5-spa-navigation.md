## Goal

The runtime-side SPA module that consumes `_testa:locationchange` events fired by the loader's monkey-patch. Coalesces bursts with a 50 ms debounce, computes canonical URLs, decides whether the URL change is meaningful, and re-runs the experiment cycle when it is.

## Context

The loader (3.1) installs the history monkey-patch and dispatches `_testa:locationchange` as a microtask after each navigation. This task consumes that event.

The 3.6 pixel has a known but uncharacterized bug around redirect query-param handling in Next.js SPAs (see memory pointer above). This task is where the fix lands. Design defensively against the failure modes enumerated there.

## Acceptance criteria

- Listener registered for `_testa:locationchange` (CustomEvent dispatched by loader).
- `popstate` and `hashchange` are also wired through this same handler.
- 50 ms debounce coalesces double-fires (Next.js `replaceState` then `pushState`, React 18 transitions).
- `canonicalize(url)` strips `_testa_*` query params and sorts query keys; same canonical URL = no re-eval.
- On meaningful URL transition: re-fire `page_view`, re-evaluate audience + URL rules, re-fire `experiment_view` if matched experiment changed (same experiment + same variation = no re-fire).
- Hash-only changes are gated by a per-project setting `spa.hash_routes` (default `false`); when `false`, hash changes do NOT trigger pageview/eval.
- Cross-frame edge case: only patches the top-level frame's history; iframes are left alone (documented).
- Diagnostic ring buffer of last N=20 location-change events stored on `window.__pixel_debug` for forensics.

## Implementation notes

```ts
// apps/pixel/src/runtime/spa.ts (sketch)
import { canonicalize } from './url-canonical.ts';

let lastCanonical = canonicalize(window.location.href);
let debounceTimer: ReturnType<typeof setTimeout> | null = null;

function onLocationChange(): void {
  if (debounceTimer) return;
  debounceTimer = setTimeout(() => {
    debounceTimer = null;
    const next = canonicalize(window.location.href);
    if (next === lastCanonical) return;
    lastCanonical = next;
    pushDebug({ from: lastCanonical, to: next, ts: Date.now() });
    runExperimentCycle();   // re-evaluates everything
    emitPageView();
  }, 50);
}

window.addEventListener('_testa:locationchange', onLocationChange);
window.addEventListener('popstate', onLocationChange);
window.addEventListener('hashchange', onLocationChange);
```

`canonicalize`:
- Strip `_testa_*` query params (`_testa_cd`, `_tu`, etc.).
- Strip the hash unless `spa.hash_routes` is on.
- Sort remaining query keys alphabetically.
- Lowercase the host. Keep path case-sensitive.
- Return string for cheap equality compare.

## Tests

happy-dom:
- `pushState(s, '', '/foo?b=2&a=1')` then `pushState(s, '', '/foo?a=1&b=2')` → only one re-eval (canonical URL identical).
- `pushState(s, '', '/foo?_tu=abc')` → ignored (the `_tu` param strip means same canonical as the prior URL).
- `pushState(s, '', '/foo')` then `pushState(s, '', '/bar')` within 30 ms → debounced; only one re-eval, with the latest URL.
- `popstate` triggers re-eval.
- `hashchange` does NOT trigger re-eval when `spa.hash_routes` is `false` (default).
- `hashchange` DOES trigger re-eval when `spa.hash_routes` is `true`.
- `__pixel_debug.locationChanges` accumulates entries.

## Out of scope

- The history monkey-patch itself (3.1 ships it).
- Repro harness across Next.js versions (3.11).
- `_testa.navigate(url)` public API — wired in 3.2 but its behavior is the same as a manual `pushState` (delegates here).
