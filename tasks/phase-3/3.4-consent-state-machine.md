## Goal

A small state machine tracking whether the visitor has consented. Default `granted` (matches GA4). Customer's CMP can flip via `_testa.consent('denied')` or by dispatching a `cmp:consent-changed` CustomEvent.

## Acceptance criteria

- `getState()` returns `'granted' | 'denied' | 'unknown'` with default `'granted'`.
- `setState(state)` updates and notifies subscribers; idempotent.
- `subscribe(handler)` registers a listener for state transitions.
- DOM event listener for `cmp:consent-changed` reads `event.detail` and calls `setState`.
- When `consent_mode === 'strict'` (project config), pixel waits until first `setState('granted')` before allowing any tracking call to fire (queues; drains on grant).
- Tests cover default state, transitions, `cmp:consent-changed` integration, and `strict` gating semantics.

## Implementation notes

```ts
type ConsentState = 'granted' | 'denied' | 'unknown';

let _state: ConsentState = 'granted';
const _subscribers: ((s: ConsentState) => void)[] = [];

export function getState(): ConsentState { return _state; }
export function setState(next: ConsentState): void {
  if (next === _state) return;
  _state = next;
  for (const fn of _subscribers) fn(next);
}
export function subscribe(fn: (s: ConsentState) => void): () => void {
  _subscribers.push(fn);
  return () => { /* unsubscribe */ };
}

window.addEventListener('cmp:consent-changed', (e: any) => {
  const v = e?.detail;
  if (v === 'granted' || v === 'denied' || v === 'unknown') setState(v);
});
```

Strict-mode gating: hold all tracking calls in a separate queue keyed by `state !== 'granted'`. When state flips to `'granted'`, drain.

## Tests

- Default `getState()` is `'granted'`.
- `setState('denied')` notifies subscribers.
- `cmp:consent-changed` with detail `'denied'` flips state.
- Strict mode + state `'unknown'`: `track('foo')` does NOT call transport. After `setState('granted')`: it DOES.
- Subscribe/unsubscribe lifecycle.

## Out of scope

- Actual cookie/IP rewriting on denied — that's the edge worker's job (visitor_id rotation, IP truncation).
- `_testa_uuid` cookie clearing on denied — the worker handles via `Set-Cookie` Max-Age=0.
