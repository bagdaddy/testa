## Goal

Top-level coverage check across the pixel: ≥80% statements via Vitest, plus Playwright e2e for the golden flows (load → assign → apply → track → load() resolves).

## Acceptance criteria

- Vitest `coverage:report` shows ≥80% statements / ≥75% branches.
- Playwright suite covers:
  - Cold load: pixel runs, applies a CSS variation, fires `_testa.load()`.
  - Cold load with redirect experiment: pixel redirects to variation URL, query params preserved.
  - Cold load with audience condition: pixel evaluates audience, only matching visitors get the variation.
  - SPA route change: pixel re-evaluates, re-applies if matched experiment changed.
  - Network failure: outbox retries on subsequent pageload.
  - Consent denied: tracking calls held in queue until granted.
  - Legacy listener (`Analytica.eventEmitter.on('variation_applied', ...)`): customer's existing listener fires.
- All e2e tests run against the locally-built bundle from task 3.14.

## Implementation notes

`vitest.config.ts` adds `coverage: { reporter: ['text', 'lcov'], thresholds: { lines: 80, branches: 75 } }`.

E2e fixtures are minimal HTML pages that include the locally-built pixel and exercise specific flows.

## Out of scope

- Browser combinations beyond chromium (CI cost).
- Visual regression testing (separate concern).
- 3.6 / 3.4 frozen-bundle equivalence tests (separate task; needs both bundles in the same KV namespace).
