## Goal

Comprehensive miniflare-based test that exercises the full `/track` path: pixel POSTs → enrich → bot filter → DurableObject batch → forward → mock collector receives a valid HMAC-signed batch.

## Acceptance criteria

- Vitest pool-workers with miniflare runs the worker against in-memory KV, DO, and a mock collector handler bound via service binding or a fetch interceptor.
- Test: POST /track with 50 pixel events → mock collector receives one batch of 50 events with valid HMAC.
- Test: POST /track with 5 events → wait 600ms → mock collector receives one batch of 5.
- Test: POST /track with a verifiedBot CF property → response 204, NO collector POST.
- Test: GET /projects/foo.js with KV seeded → 200 + `cfPrefill` body + Set-Cookie.
- Coverage gate: `apps/edge/src/` ≥ 80% lines and branches.

## Implementation notes

Mock collector via `vi.fn().mockImplementation(async (req) => { ... })` and bind it via `unstable_dev` or by overriding `fetch` in the test env. miniflare also supports binding a service binding to a different worker — could write a tiny mock worker for that.

The HMAC verify in the test should run the **collector's** `verifyHmac` against the worker's signed body, proving the contract closes:

```ts
import { verifyHmac } from '@testa-platform/collector/auth/hmac.ts';
// ...
const sig = capturedRequest.headers.get('x-edge-signature');
const body = await capturedRequest.text();
expect(verifyHmac(body, sig, env.INGEST_SHARED_SECRET)).toBe(true);
```

If the collector package isn't directly importable from the edge tests, copy the small `verifyHmac` function into a shared location (`packages/shared-types/src/hmac.ts`) — but only for tests; runtime keeps `node:crypto` (collector) and `crypto.subtle` (edge) separate.

## Out of scope

- Performance / load testing.
- Actual CF deployment (Phase 2.9).
- E2E from a real browser (Phase 3.10 covers that against the real edge).
