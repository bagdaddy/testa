## Goal

A Cloudflare DurableObject that buffers `EnrichedEvent` items, flushes on either a 500ms alarm or 50-event threshold, and POSTs the batch to the collector via the ingest module (Phase 2.6 will provide the HMAC sign + POST logic).

For Phase 2.5 only: implement the buffering + alarm logic. The flush calls a stub `flushFn` injected by the host that just resolves; Phase 2.6 replaces the stub with real ingest.

## Acceptance criteria

- One DO per request? **No** — one DO per project/host (use the request's `Host` header as the DO id namespace). Single hot DO per customer is fine for v1; if a customer is huge, scale by sharding key (post-pilot).
- `add(event)` returns immediately. Internal buffer grows.
- If buffer reaches 50 events: flush immediately.
- Else: ensure an alarm exists at `now + 500ms`. On alarm: flush.
- `flush()` calls `flushFn(events)`; on success clears the buffer; on error, retains events for the next flush attempt with exp backoff (500ms, 1s, 2s, max 8s).
- DO storage NOT used (in-memory only). Events lost on DO eviction are acceptable at this scale; alternative is `state.storage.put` per add which is too expensive.
- 80%+ coverage. Use `@cloudflare/vitest-pool-workers` for DO tests.

## Implementation notes

```ts
// apps/edge/src/batch.ts (sketch)
import type { EnrichedEvent } from '@testa-platform/shared-types';

export type FlushFn = (events: EnrichedEvent[]) => Promise<void>;

const FLUSH_AFTER_MS = 500;
const FLUSH_AT_COUNT = 50;
const MAX_BACKOFF_MS = 8_000;

export class BatchBuffer implements DurableObject {
  private buffer: EnrichedEvent[] = [];
  private flushing = false;
  private backoff = 0;

  constructor(
    private readonly state: DurableObjectState,
    private readonly env: Env,
  ) {}

  async fetch(request: Request): Promise<Response> {
    const op = new URL(request.url).pathname;
    if (op === '/add') {
      const event = (await request.json()) as EnrichedEvent;
      await this.add(event);
      return new Response(null, { status: 204 });
    }
    return new Response('not found', { status: 404 });
  }

  async add(event: EnrichedEvent): Promise<void> {
    this.buffer.push(event);
    if (this.buffer.length >= FLUSH_AT_COUNT) {
      await this.flush();
      return;
    }
    const existing = await this.state.storage.getAlarm();
    if (existing == null) {
      await this.state.storage.setAlarm(Date.now() + FLUSH_AFTER_MS);
    }
  }

  async alarm(): Promise<void> {
    await this.flush();
  }

  private async flush(): Promise<void> {
    if (this.flushing || this.buffer.length === 0) return;
    this.flushing = true;
    const batch = this.buffer.splice(0, this.buffer.length);
    try {
      await this.flushFn()(batch);
      this.backoff = 0;
    } catch (err) {
      // restore + schedule retry
      this.buffer.unshift(...batch);
      this.backoff = Math.min(this.backoff === 0 ? 500 : this.backoff * 2, MAX_BACKOFF_MS);
      await this.state.storage.setAlarm(Date.now() + this.backoff);
      console.error('[batch] flush failed; retry in', this.backoff, err);
    } finally {
      this.flushing = false;
    }
  }

  private flushFn(): FlushFn {
    // Phase 2.5: stub. Phase 2.6 swaps in real HMAC + POST.
    // Return a function that captures `this.env` for the real impl.
    return async (events) => {
      // Imported in Phase 2.6: forwardBatch(events, this.env)
      console.log('[batch] would flush', events.length);
    };
  }
}
```

## Tests

`__tests__/batch.test.ts` (using miniflare DO support):

- `add(event)` x 49 → no flush yet.
- `add(event)` x 50 → flush called once with 50 events.
- `add(event)` x 5 + wait 600ms (advance test clock) → flush called with 5 events.
- Flush throws → events restored to buffer; alarm rescheduled.
- 5 consecutive flush failures → backoff caps at 8s.

## Out of scope

- HMAC sign + actual POST to collector (Phase 2.6).
- Per-project rate limiting.
- DO sharding for high-volume customers.
- Persistence to DO storage (we accept in-memory loss on eviction).
