## Goal

Deploy the edge worker to a Cloudflare staging zone (`testa-edge-staging.workers.dev` or similar). **The autonomous routine MUST PAUSE here.** Real CF account access, KV namespace creation, and secret setting are explicit-human-approval steps per `AGENTS.md`.

## Acceptance criteria — PRE-PAUSE

- `wrangler deploy --dry-run` succeeds locally.
- A pre-deploy checklist PR is open, summarizing:
  - What KV namespaces need to be created (`KV_PROJECT_CONFIG`, `KV_INTEGRATION_BUNDLES`).
  - What secrets need to be set (`INGEST_SHARED_SECRET`, `INGEST_ORIGIN_URL`, `COOKIE_FALLBACK_DOMAIN`, `VISITOR_ID_SALT`).
  - The seed data needed in KV before first deploy (the legacy `integration_bundle:3.4` and `integration_bundle:3.6` text from crobot).
  - The smoke test plan (curl `/health`, curl signed `/track` with one fake event, watch the collector for it).

## Acceptance criteria — POST-HUMAN-APPROVAL (NOT for the routine)

- Real KV IDs filled into `wrangler.toml` (or moved to `wrangler.staging.toml`).
- Secrets set via `wrangler secret put`.
- `wrangler deploy --env staging` clean.
- Smoke test passes.

## Implementation notes

The routine should:

1. Run `wrangler deploy --dry-run --env staging` to verify the build is deploy-ready.
2. Open a PR titled `chore(edge): 2.9 staging deploy checklist (READY FOR HUMAN)` containing only the checklist doc.
3. **Stop. Do not deploy.**

A human reviews, fills in real KV IDs + secrets, runs `wrangler deploy --env staging`, runs the smoke test, and merges the PR with a "deployed by <human> on <date>" comment.

## Out of scope

- Production deploy (separate task in Phase 6).
- Real customer DNS / CNAME setup (Phase 7).
