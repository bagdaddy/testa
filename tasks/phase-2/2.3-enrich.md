## Goal

A pure function `enrich(request, pixelEvent)` → `EnrichedEvent` that adds `country`, `region`, `device_type`, `browser`, `os`, `ingested_at`. Plus IP truncation for denied-consent rows. No bot filtering yet — that's task 2.4 and runs after enrich.

## Acceptance criteria

- `enrich(request, pixelEvent)` returns the enriched shape per `packages/shared-types/src/event.ts`.
- Reads `cf-ipcountry` for country (default `'XX'`).
- Reads CF region if available (default `''`).
- Parses `user-agent` via `ua-parser-js` for browser + os + device family.
- `device_type` mapping: `mobile|tablet|desktop` from UA; `'unknown'` if UA empty/garbled. (`'bot'` set by bot filter, not here.)
- Truncates IP per consent state:
  - `granted` / `unknown`: don't pass IP into the enriched event at all (CH never sees it).
  - `denied`: also don't pass IP, but the visitor_id rotation logic uses the truncated IP internally to derive the daily ephemeral visitor_id.
- `ingested_at = Date.now()`.
- 90%+ coverage; table-driven tests for UA parsing.

## Implementation notes

- `ua-parser-js` is the standard. Add to `apps/edge/package.json` deps.
- The IP truncation: `ipv4: '203.0.113.42' → '203.0.113.0'` (drop last octet). `ipv6: 'a:b:c:d:e:f:g:h' → 'a:b:c::'` (drop last 80 bits / 5 groups).
- The truncated IP and visitor_id are needed for **denied-consent visitor_id rotation**, which lives in `cookies.ts`. Export a helper `truncateIp(ip: string)` from `enrich.ts` and have cookies.ts import it.

```ts
// apps/edge/src/enrich.ts (sketch)
import { UAParser } from 'ua-parser-js';
import type { PixelEvent, EnrichedEvent } from '@testa-platform/shared-types';

export function enrich(request: Request, ev: PixelEvent): EnrichedEvent {
  const ua = request.headers.get('user-agent') ?? '';
  const parser = new UAParser(ua);
  const result = parser.getResult();

  return {
    ...ev,
    ingested_at: Date.now(),
    country: request.headers.get('cf-ipcountry')?.toUpperCase() ?? 'XX',
    region: (request as any).cf?.region ?? '',
    device_type: mapDeviceType(result.device.type),
    browser: result.browser.name ?? '',
    os: result.os.name ? `${result.os.name} ${result.os.version ?? ''}`.trim() : '',
    is_bot: 0, // set by bot filter in 2.4
  };
}

export function truncateIp(ip: string): string {
  if (ip.includes('.')) {
    const parts = ip.split('.');
    return parts.slice(0, 3).join('.') + '.0';
  }
  if (ip.includes(':')) {
    const parts = ip.split(':');
    return parts.slice(0, 3).join(':') + '::';
  }
  return '';
}

function mapDeviceType(t: string | undefined): EnrichedEvent['device_type'] {
  if (t === 'mobile' || t === 'tablet') return t;
  if (!t) return 'desktop'; // ua-parser leaves desktop empty
  return 'unknown';
}
```

## Tests

`__tests__/enrich.test.ts`:

- iOS Safari UA → `device_type='mobile', browser='Safari', os='iOS ...'`.
- Android Chrome UA → `device_type='mobile', browser='Chrome', os='Android ...'`.
- Mac Chrome UA → `device_type='desktop', browser='Chrome', os='Mac OS ...'`.
- Empty UA → `device_type='unknown', browser='', os=''`.
- `cf-ipcountry: GB` → `country='GB'`.
- Missing `cf-ipcountry` → `country='XX'`.
- `truncateIp('203.0.113.42')` → `'203.0.113.0'`.
- `truncateIp('2001:db8:abcd:ef12:1234:5678:90ab:cdef')` → `'2001:db8:abcd::'`.
