## Goal

Split the existing single-file `apps/edge/src/index.ts` Hono app into per-route modules so subsequent tasks (cookies, enrich, batch, ingest forward, KV serve) each own a small focused file. Routes still return 501 for unimplemented logic; this task is purely structural.

## Acceptance criteria

- `src/routes/health.ts` exports a `Hono` sub-app with `GET /health`.
- `src/routes/track.ts` exports a `Hono` sub-app with `POST /track` (501) and `OPTIONS /track` (204 + CORS).
- `src/routes/serve.ts` exports a `Hono` sub-app with `GET /projects/:slug{.+\\.js}` (returns the placeholder JS).
- `src/index.ts` mounts all three.
- Existing smoke test in `src/__tests__/smoke.test.ts` keeps passing.
- New smoke test added: `GET /projects/anything.js` returns 200 with `content-type: application/javascript`.

## Implementation notes

- Hono `app.route('/', subApp)` to mount.
- Keep `Env` type in `src/types.ts` (move it out of `index.ts`) so route modules can import it without circular deps.
- The `BatchBuffer` DurableObject stub stays in `index.ts` (it's bound by name; moving it is fine but not necessary for v1).

## Tests

- Existing `smoke.test.ts` — must still pass.
- Add `src/routes/__tests__/serve.test.ts` for the `/projects/foo.js` placeholder response.

## Out of scope

- Any logic — just structural split.
- KV reads / writes (Phase 2.7).
- Cookies (Phase 2.2).
