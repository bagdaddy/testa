## Goal

A `botSignal(request, enrichedEvent)` function that returns `{ is_bot: 0 | 1, score: number, reasons: string[] }`. Uses only **free** CF signals — paid Bot Management is out of scope. Decisions go into the event written to CH (`is_bot` column). The worker uses the result to optionally drop or downsample, but for v1 it just tags.

## Acceptance criteria

- Drops events where `cf.botManagement.verifiedBot === true` (Googlebot, Bingbot, etc.). Returns `{ is_bot: 1, reasons: ['verifiedBot'] }`.
- Marks `is_bot=1` for these heuristic hits (any single hit is enough to mark, not drop):
  - UA matches `/HeadlessChrome|PhantomJS|Selenium|playwright|puppeteer/i`.
  - `accept-language` header missing or empty.
  - `user-agent` empty (rare; almost certainly bot or curl).
  - ASN matches the bad-list (`data/asn-bad-list.ts`).
- Score is a 0–100 informational number (sum of weights). Used for diagnostics; not the decision.
- For `verifiedBot`, the worker DROPS the event entirely (returns 204 to keep things looking healthy but doesn't forward to collector).
- For other heuristic hits, the worker FORWARDS with `is_bot=1`. Dashboards can filter.
- 90%+ coverage; table-driven tests with realistic UAs.

## Implementation notes

- ASN bad list: ship a hand-curated list of ~20-50 ASNs known for hosting bots / scrapers (DigitalOcean, OVH, AWS public ranges, etc.). Don't be aggressive — false positives hurt revenue measurement. Source: pick the top 10 from public bot-traffic studies; conservative.

```ts
// apps/edge/src/data/asn-bad-list.ts
export const BOT_ASN_BLOCKLIST = new Set<number>([
  14061,   // DigitalOcean
  16509,   // Amazon AWS
  14618,   // Amazon AWS (different)
  16276,   // OVH
  // ... add ~10-20 more, conservatively
]);
```

```ts
// apps/edge/src/bot.ts (sketch)
import { BOT_ASN_BLOCKLIST } from './data/asn-bad-list.ts';

export interface BotResult {
  is_bot: 0 | 1;
  score: number;          // 0..100, informational
  reasons: string[];
  drop: boolean;          // true if we should not forward to collector
}

const HEADLESS = /HeadlessChrome|PhantomJS|Selenium|playwright|puppeteer/i;

export function botSignal(request: Request): BotResult {
  const reasons: string[] = [];
  let score = 0;
  let drop = false;

  const cf = (request as any).cf ?? {};
  if (cf.botManagement?.verifiedBot) {
    return { is_bot: 1, score: 100, reasons: ['verifiedBot'], drop: true };
  }

  const ua = request.headers.get('user-agent') ?? '';
  if (!ua) {
    reasons.push('empty_ua'); score += 80;
  }
  if (HEADLESS.test(ua)) {
    reasons.push('headless'); score += 60;
  }
  if (!request.headers.get('accept-language')) {
    reasons.push('no_accept_language'); score += 30;
  }
  const asn = cf.asn as number | undefined;
  if (asn != null && BOT_ASN_BLOCKLIST.has(asn)) {
    reasons.push(`bad_asn:${asn}`); score += 50;
  }

  return { is_bot: score >= 30 ? 1 : 0, score, reasons, drop };
}
```

## Tests

`__tests__/bot.test.ts`:

- `verifiedBot=true` → `drop=true, is_bot=1`.
- Empty UA → `is_bot=1, score≥80`.
- Headless UA → `is_bot=1, score≥60`.
- Missing accept-language alone → `is_bot=0` (not enough to flip; score=30 is threshold-borderline; pick 30 weights so score=30 → is_bot=1).
- Bad ASN alone → `is_bot=1, score≥50`.
- Normal Mac Chrome with full headers and good ASN → `is_bot=0, score=0`.
- Combination (headless + empty UA) → `is_bot=1, score≥140` (cap at 100 in display).

## Out of scope

- Paid CF Bot Management score.
- Behavioral signals (mouse movement, viewport size). The pixel could send these in `props`, but v1 doesn't.
- ML / heuristic learning. Static rules only.
- Per-customer bot-list overrides.
